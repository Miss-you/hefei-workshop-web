断点策略
断点 宽度 布局 典型设备
默认 < 640px 单列布局 手机
sm ≥ 640px 双列布局 平板竖屏
lg ≥ 1024px 多列布局 桌面端

AI Prompt
Prompt 响应式改造
请为购物车和结算页添加响应式布局：

1. 移动端优先策略
2. 断点：默认(手机) -> sm:640px(平板) -> lg:1024px(桌面)
3. 商品网格：1列 -> 2列 -> 4列
4. 结算页：堆叠布局 -> 并排布局
5. 按钮和表单元素触控友好（最小 44px）
原理：为什么是移动端优先？
渐进增强：从最小屏幕开始，逐步添加复杂布局，确保基础体验可用。

代码更简洁：基础样式无前缀，只在需要时添加响应式类。

性能更好：移动端不加载桌面端的复杂样式，CSS 解析更快。

符合用户习惯：移动流量占比超过 60%，优先保证主要用户体验。
最佳实践 vs 不建议
最佳实践
  • 移动端优先，逐步增强
  • 使用 Tailwind 断点前缀
  • 测试真实设备效果
  • 触控目标至少 44px
  • 图片使用响应式尺寸
不建议
  • 桌面优先然后隐藏元素
  • 硬编码固定宽度
  • 只在浏览器模拟测试
  • 按钮太小难以点击
  • 忽略横屏模式
验收清单
  • 结算页显示订单摘要和总价
  • 移动端（< 640px）单列布局正常
  • 平板端（≥ 640px）双列布局正常
  • 桌面端（≥ 1024px）多列布局正常
  • 所有按钮触控友好
